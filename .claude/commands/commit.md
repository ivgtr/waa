コミットを作成する前に、以下の品質チェックを順番に実行してください。

## 事前チェック

1. `npm run typecheck` を実行し、型エラーがないことを確認
2. `npm run lint` を実行し、lint エラーがないことを確認

いずれかが失敗した場合はコミットを中止し、エラー内容を報告してください。

## コミット作成

事前チェックがすべて通った場合のみ、以下の方針でコミットを作成してください。

- 変更を意味単位（機能追加、バグ修正、リファクタリング等）に分割し、それぞれ個別のコミットにする
- 既存のコミット履歴のスタイル（prefix、文体）を確認し、それに合わせたメッセージを書く
- 無関係な変更を1つのコミットに混ぜない
